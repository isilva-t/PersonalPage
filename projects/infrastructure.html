Multi-Service Infrastructure

Feb 2025 - Apr 2025

Containerized web infrastructure project as part of 42 Porto curriculum. Deployed multi-service architecture with NGINX, WordPress, and MariaDB using Docker, enforcing security protocols and data persistence under strict configuration requirements.

Infrastructure Components

  • • NGINX: Reverse proxy with SSL/TLS encryption and custom configuration
  • • WordPress: CMS with PHP-FPM processing and optimized performance
  • • MariaDB: Relational database with persistent volume management
  • • Docker Network: Isolated network with service-to-service communication
  • • Security: Environment variables, secret management, and hardened configurations
  • • Persistence: Volume mounting for database and application data survival

Technical Challenge: Built entire infrastructure from Dockerfiles (no pre-built images allowed). Configured services to communicate securely through internal network while exposing only necessary ports. Implemented proper volume management to ensure data persistence across container restarts.
